# 安全检查清单

## 风险等级定义

| 等级 | 分值 | 含义 |
|------|------|------|
| 🔴 高风险 | ≥20 | 需立即审查 |
| 🟡 中风险 | 5-19 | 建议关注 |
| 🟢 低风险 | <5 | 正常范围 |

## 检测类别

### 高危
- **数据外泄**：向外部 URL 发送 POST/PUT 请求
- **凭证窃取**：访问 .ssh、.pem、.env、.aws/ 等
- **系统破坏**：`rm -rf /`、`format C:` 等危险删除
- **权限提升**：sudo、chmod 777、setuid

### 中危
- **动态代码执行**：eval、exec、subprocess(shell=True)
- **网络扫描**：nmap、masscan
- **代码混淆**：base64 编码 + 执行

### 低危
- **环境变量读取**：os.environ（合法但需关注）
- **文件遍历**：os.walk / glob（项目扫描常用）

## 白名单

以下模式在特定上下文属合法使用：
- `os.environ`：读取配置、获取路径
- `subprocess`：CLI 工具封装
- `os.walk`：本地项目扫描
- `base64`：API 请求编码

## 处理建议

1. **高风险**：立即审查对应代码行，确认是否为预期行为
2. **中危**：检查是否有合理的业务需求
3. **低危**：一般无需处理，了解即可
4. **误报**：安全审计基于模式匹配，合法代码可能触发告警，需人工判断
